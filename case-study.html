Overview

myFlix is a comprehensive web application built on the MERN (MongoDB, Express, React, Node.js) stack, designed to offer users a seamless experience in accessing extensive movie-related information. It allows users to register accounts, manage personal details, and curate a personalized list of favorite movies.

Purpose

myFlix was a project I independently developed during my web development course at CareerFoundry, intended to showcase my proficiency in full-stack JavaScript development.

Objective

The goal of the project was to create a comprehensive full-stack application that could serve as a valuable addition to my professional portfolio. My primary objective was to construct both the server-side and client-side of the application entirely from the ground up.

Approach

Server-Side

I developed a RESTful API with Node.js and Express, which communicates with a non-relational MongoDB database. The API is accessible through standard HTTP methods such as GET or POST. It utilizes CRUD methods to retrieve and store data in the database, enabling the provision of movie information in JSON format.

Client-Side

After completing the API, I started to build the interface users would need when making requests to, and receiving responses from, the server-side. It is a single-page, responsive application, developed with React and React-Redux. It provides several interface views, including, but not limited to, a main view (shows a list of all movies), single movie view (shows data about a single movie and allows users to add the movie to their list of favorites), a login view, a registration view, and a profile view (where users can update their user data and list of favorites).

Challenges

Challenge: Complex API Integration: Integrating the RESTful API with the client-side application posed several technical challenges, including data synchronization issues and API endpoint complications.

Solution: To address these issues, I conducted extensive testing and debugging to identify the root causes of the integration problems. Implementing appropriate error handling mechanisms and thoroughly reviewing the API documentation helped streamline the integration process and ensure the seamless communication between the client-side and server-side components

Duration

Throughout the duration of the project, I invested approximately 2 months to complete myFlix. The first month was dedicated to learning server-side programming and exploring the intricacies of Node.js. During this time, I also delved into building the API, ensuring its robustness and reliability. The second month revolved around mastering client-side programming and delving into the intricacies of React. Employing a Node.js client-side approach, I successfully finalized the project, culminating in a comprehensive full-stack web application that seamlessly integrates server and client functionalities.
